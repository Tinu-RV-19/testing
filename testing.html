Data Testing

1. Data Validation Testing

Ensures that the data follows required formats and rules.

2. Data Completeness Testing

Checks if important values are missing.

3. Data Accuracy Testing

Ensures that the data reflects real-world values.

4. Data Consistency Testing

Verifies that data is consistent across multiple systems.

5. Duplicate Data Testing

Identifies duplicate records in the dataset.

6. Data Integrity Testing

Ensures relationships between datasets remain valid.
